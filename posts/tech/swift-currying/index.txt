13:T35ef,<p>This topic (aka Currying) has been mentioned as a part of Functional programming. Though it’s kind of fun to use currying, we often see it on the list of <em>what-must-not-be-named</em> (along with functors, applicative, monads...) LOL. In this article, I’ll show you a couple of ways to have fun with currying.</p>
<h3 id="what-is-currying">What is currying?</h3>
<p>Roughly speaking, currying is the act of <em><strong>translating a single function of N arguments into N functions of a single argument</strong></em>. For those who did not know what it is, take a look at <a href="https://www.objc.io/blog/2014/11/10/functional-snippet-6-currying/">this link</a>.
Suppose we a have a function multiply:</p>
<pre><code class="hljs language-swift"><span class="hljs-keyword">func</span> <span class="hljs-title function_">mutiply</span>(<span class="hljs-params">op1</span>: <span class="hljs-type">Int</span>, <span class="hljs-params">op2</span>: <span class="hljs-type">Int</span>) -> <span class="hljs-type">Int</span> {
   <span class="hljs-keyword">return</span> op1 <span class="hljs-operator">*</span> op2
}
</code></pre>
<p>The type of multiply is: <code>(Int, Int) -> Int</code>.
Currying this function results in another function which is of type: <code>Int -> Int -> Int</code>. And this is how to get the result using the curried function:</p>
<pre><code class="hljs language-swift"><span class="hljs-keyword">let</span> result <span class="hljs-operator">=</span> curriedMultiply(<span class="hljs-number">2</span>)(<span class="hljs-number">3</span>)
</code></pre>
<p>Note that the result of <code>curriedMultiply(2)</code> is a function <code>Int -> Int</code>. This function take an integer <code>x</code> as a parameter and return <code>2 * x</code>, equivalent to making a number double. Similarly, we could create lots of derived functions:</p>
<pre><code class="hljs language-swift"><span class="hljs-keyword">let</span> makeDouble <span class="hljs-operator">=</span> curriedMultiply(<span class="hljs-number">2</span>)		<span class="hljs-comment">// Int -> Int</span>
<span class="hljs-keyword">let</span> makeTriple <span class="hljs-operator">=</span> curriedMultiply(<span class="hljs-number">3</span>)		<span class="hljs-comment">// Int -> Int</span>
<span class="hljs-keyword">let</span> makeFourTimes <span class="hljs-operator">=</span> curriedMultiply(<span class="hljs-number">4</span>)	<span class="hljs-comment">// Int -> Int</span>
</code></pre>
<h3 id="whats-the-difference">What's the difference?</h3>
<p>We transform the values before passing them as the parameters of a function on a very regular basis. In other words, we often use a function like this:</p>
<pre><code class="hljs language-swift"><span class="hljs-keyword">let</span> result <span class="hljs-operator">=</span> f(g(a), h(b), k(c))
</code></pre>
<p>In this scenario, isolating the parameters make the function more flexible &#x26; reusable.</p>
<h3 id="examples">Examples</h3>
<h4 id="write-function-curry">Write function "curry"</h4>
<p>First of all, lets write a function to curry a function:</p>
<pre><code class="hljs language-swift"><span class="hljs-keyword">func</span> <span class="hljs-title function_">curry</span>(<span class="hljs-params">f</span>: (<span class="hljs-type">A</span>, <span class="hljs-type">B</span>) -> <span class="hljs-type">R</span>) -> <span class="hljs-type">A</span> -> <span class="hljs-type">B</span> -> <span class="hljs-type">R</span> {
	<span class="hljs-keyword">return</span> { a <span class="hljs-keyword">in</span> { b <span class="hljs-keyword">in</span> f(a, b) } }
}
</code></pre>
<p>Similarly, you could write more implementations of curry for functions that have 3, 4, 5 parameters</p>
<pre><code class="hljs language-swift"><span class="hljs-keyword">func</span> <span class="hljs-title function_">curry</span>(<span class="hljs-params">f</span>: (<span class="hljs-type">A</span>, <span class="hljs-type">B</span>, <span class="hljs-type">C</span>) -> <span class="hljs-type">R</span>) -> <span class="hljs-type">A</span> -> <span class="hljs-type">B</span> -> <span class="hljs-type">C</span> -> <span class="hljs-type">R</span>
<span class="hljs-keyword">func</span> <span class="hljs-title function_">curry</span>(<span class="hljs-params">f</span>: (<span class="hljs-type">A</span>, <span class="hljs-type">B</span>, <span class="hljs-type">C</span>, <span class="hljs-type">D</span>) -> <span class="hljs-type">R</span>) -> <span class="hljs-type">A</span> -> <span class="hljs-type">B</span> -> <span class="hljs-type">C</span> -> <span class="hljs-type">D</span> -> <span class="hljs-type">R</span>
<span class="hljs-keyword">func</span> <span class="hljs-title function_">curry</span>(<span class="hljs-params">f</span>: (<span class="hljs-type">A</span>, <span class="hljs-type">B</span>, <span class="hljs-type">C</span>, <span class="hljs-type">D</span>, <span class="hljs-type">E</span>) -> <span class="hljs-type">R</span>) -> <span class="hljs-type">A</span> -> <span class="hljs-type">B</span> -> <span class="hljs-type">C</span> -> <span class="hljs-type">D</span> -> <span class="hljs-type">E</span> -> <span class="hljs-type">R</span>
</code></pre>
<p>Now, lets curry a function</p>
<pre><code class="hljs language-swift"><span class="hljs-keyword">func</span> <span class="hljs-title function_">addWithScale</span>(<span class="hljs-params">op1</span>: <span class="hljs-type">Int</span>, <span class="hljs-params">op2</span>: <span class="hljs-type">Int</span>, <span class="hljs-params">scale</span>: <span class="hljs-type">Int</span>) -> <span class="hljs-type">Int</span> {
	<span class="hljs-keyword">return</span> (op1 <span class="hljs-operator">+</span> op2) <span class="hljs-operator">*</span> scale
}

<span class="hljs-keyword">let</span> curriedAddWithScale <span class="hljs-operator">=</span> curry(addWithScale)
<span class="hljs-keyword">let</span> result1 <span class="hljs-operator">=</span> curriedAddWithScale(<span class="hljs-number">1</span>)(<span class="hljs-number">2</span>)(<span class="hljs-number">3</span>)		<span class="hljs-comment">// result1 = 9</span>
<span class="hljs-keyword">let</span> scaleMySum <span class="hljs-operator">=</span> curriedAddWithScale(<span class="hljs-number">1</span>)(<span class="hljs-number">2</span>)		<span class="hljs-comment">// scaleMySum: Int -> Int</span>
<span class="hljs-keyword">let</span> result2 <span class="hljs-operator">=</span> scaleMySum(<span class="hljs-number">3</span>)						<span class="hljs-comment">// result2 = 9</span>
</code></pre>
<h4 id="use-with-custom-operators-">Use with custom operators <code>|></code></h4>
<p>To make the code more readable, lets define an operator <code>|></code> (we read it as "apply") that calculates the result of a specific function:</p>
<pre><code class="hljs language-swift"><span class="hljs-keyword">infix</span> <span class="hljs-keyword">operator</span> <span class="hljs-title">|></span> { associativity left }
<span class="hljs-keyword">func</span> <span class="hljs-title function_">|></span> &#x3C;<span class="hljs-type">A</span>, <span class="hljs-type">R</span>>(<span class="hljs-params">f</span>: <span class="hljs-type">A</span> -> <span class="hljs-type">R</span>, <span class="hljs-params">x</span>: <span class="hljs-type">A</span>) -> <span class="hljs-type">R</span> {
    <span class="hljs-keyword">return</span> f(x)
}
</code></pre>
<p>Now, use it with <code>curriedAddWithScale</code>:</p>
<pre><code class="hljs language-swift"><span class="hljs-keyword">let</span> result1 <span class="hljs-operator">=</span> curriedAddWithScale <span class="hljs-operator">|></span> <span class="hljs-number">1</span> <span class="hljs-operator">|></span> <span class="hljs-number">2</span> <span class="hljs-operator">|></span> <span class="hljs-number">3</span>
<span class="hljs-keyword">let</span> scaleMySum <span class="hljs-operator">=</span> curriedAddWithScale <span class="hljs-operator">|></span> <span class="hljs-number">1</span> <span class="hljs-operator">|></span> <span class="hljs-number">2</span>
<span class="hljs-keyword">let</span> result2 <span class="hljs-operator">=</span> scaleMySun <span class="hljs-operator">|></span> <span class="hljs-number">3</span> 
</code></pre>
<p>Still the same? Hold on, you could add some argument labels like this:</p>
<pre><code class="hljs language-swift"><span class="hljs-keyword">let</span> result1 <span class="hljs-operator">=</span> curriedAddWithScale
	<span class="hljs-operator">|></span> (op1: <span class="hljs-number">1</span>)			<span class="hljs-comment">// You could omit the label "op1" if you want</span>
	<span class="hljs-operator">|></span> (op2: <span class="hljs-number">2</span>)			<span class="hljs-comment">// You could omit the label "op2" if you want</span>
	<span class="hljs-operator">|></span> (scale: <span class="hljs-number">3</span>)		<span class="hljs-comment">// You could omit the label "scale" if you want</span>
</code></pre>
<h4 id="use-with-apple-swift-functions">Use with Apple Swift functions</h4>
<p>Have a look at the init methods of UIColor. How to get the right init function. We only want to curry this init method:</p>
<pre><code class="hljs language-swift"><span class="hljs-type">UIImage</span>.<span class="hljs-keyword">init</span>(red: <span class="hljs-type">CGFloat</span>, green: <span class="hljs-type">CGFloat</span>, blue: <span class="hljs-type">CGFloat</span>, alpha: <span class="hljs-type">CGFloat</span>)
</code></pre>
<p>Try this:</p>
<pre><code class="hljs language-swift"><span class="hljs-keyword">let</span> curriedColorInit <span class="hljs-operator">=</span> curry(<span class="hljs-type">UIColor</span>.<span class="hljs-keyword">init</span>(red:green:blue:alpha:))
<span class="hljs-keyword">let</span> orangeWithAlpha <span class="hljs-operator">=</span> curriedColorInit <span class="hljs-operator">|></span> <span class="hljs-number">0.945</span> <span class="hljs-operator">|></span> <span class="hljs-number">0.353</span> <span class="hljs-operator">|></span> <span class="hljs-number">0.133</span>
<span class="hljs-keyword">let</span> orange1 <span class="hljs-operator">=</span> orangeWithAlpha <span class="hljs-operator">|></span> <span class="hljs-number">0.9</span>
<span class="hljs-keyword">let</span> orange2 <span class="hljs-operator">=</span> orangeWithAlpha <span class="hljs-operator">|></span> <span class="hljs-number">0.3</span>

</code></pre>
<h4 id="refactor-closure">Refactor closure</h4>
<p>Suppose we have a function fetchData as follows:</p>
<pre><code class="hljs language-swift"><span class="hljs-keyword">func</span> <span class="hljs-title function_">fetchData</span>(<span class="hljs-params">path</span>: <span class="hljs-type">String</span>, <span class="hljs-params">successfulHandler</span>: <span class="hljs-type">NSData</span>? -> <span class="hljs-type">Void</span>, <span class="hljs-params">failedHandler</span>: <span class="hljs-type">NSError</span>? -> <span class="hljs-type">Void</span>) {
	<span class="hljs-operator">...</span>
}
</code></pre>
<p>I believe you have been a bit frustrated with this sort of functions because of the ugly caller like this:</p>
<pre><code class="hljs language-swift">fetchData(<span class="hljs-string">"www.mysite.com"</span>, successfulHandler: { data <span class="hljs-keyword">in</span>
    <span class="hljs-comment">// Handle success</span>
    }, failedHandler: { error <span class="hljs-keyword">in</span>
        <span class="hljs-comment">// Handle failure</span>
})
</code></pre>
<p>or this</p>
<pre><code class="hljs language-swift">fetchData(<span class="hljs-string">"www.mysite.com"</span>, successfulHandler: { data <span class="hljs-keyword">in</span>
    <span class="hljs-comment">// Handle success</span>
    }) { error <span class="hljs-keyword">in</span>
        <span class="hljs-comment">// Handle failure</span>
}

</code></pre>
<p>The <code>successfulHandler</code> and <code>failedHandler</code> should have the same indent, right? Moreover, the syntax to write the path and successfulHandler in the same line makes us hard to read.</p>
<p>Now, with the combination of curry and operator |> we could write it in such a readable manner like this:</p>
<pre><code class="hljs language-swift"><span class="hljs-keyword">let</span> fetchDataWithHandlers <span class="hljs-operator">=</span> curry(fetchData) <span class="hljs-operator">|></span> <span class="hljs-string">"www.mysite.com"</span>
fetchDataWithHandlers
    <span class="hljs-operator">|></span> (successful: { data <span class="hljs-keyword">in</span>
        <span class="hljs-comment">// Handle success</span>
    })
    <span class="hljs-operator">|></span> (failed: { error <span class="hljs-keyword">in</span>
        <span class="hljs-comment">// Handle failure</span>
    })
</code></pre>
<pre><code class="hljs language-swift"><span class="hljs-keyword">let</span> curriedAnimate <span class="hljs-operator">=</span> curry(<span class="hljs-type">UIView</span>.animateWithDuration(<span class="hljs-keyword">_</span>:animations:completion:))
curriedAnimate
    <span class="hljs-operator">|></span> (duration: <span class="hljs-number">0.25</span>)
    <span class="hljs-operator">|></span> (animation: { <span class="hljs-keyword">_</span> <span class="hljs-keyword">in</span>
        <span class="hljs-comment">// Animation here</span>
    })
    <span class="hljs-operator">|></span> (completion: <span class="hljs-literal">nil</span>)
</code></pre>
<p>It seems better looking now!</p>
<p>Have fun!</p>
