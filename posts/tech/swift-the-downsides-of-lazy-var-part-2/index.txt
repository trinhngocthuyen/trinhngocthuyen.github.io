13:T2598,<p>Continue from the <a href="/tech/swift-the-downsides-of-lazy-var-part-1">previous post</a>, we will look into a few problems when working with lazy var.</p>
<h3 id="problems">Problems</h3>
<h4 id="problem-1-not-working-smoothly-with-structs">Problem 1: not working smoothly with structs</h4>
<p>Since the getter of lazy var is mutating, using it with structs sometimes requires extra work. For example, this code below does not compile:</p>
<pre><code class="hljs language-swift"><span class="hljs-keyword">struct</span> <span class="hljs-title class_">Person</span> {
	<span class="hljs-keyword">private</span> <span class="hljs-keyword">lazy</span> <span class="hljs-keyword">var</span> name <span class="hljs-operator">=</span> <span class="hljs-string">"thuyen"</span>
	<span class="hljs-keyword">var</span> alias: <span class="hljs-type">String</span> { <span class="hljs-keyword">return</span> name } 	<span class="hljs-comment">// error: Cannot use mutating getter on immutable</span>
}
</code></pre>
<p>To make this code work, we have to explicitly declare <code>getter</code> of <code>alias</code> as <code>mutating</code>. But personally, I would rather not make an object mutable just in order to access a specific property.</p>
<pre><code class="hljs language-swift"><span class="hljs-keyword">struct</span> <span class="hljs-title class_">Person</span> {
	<span class="hljs-keyword">private</span> <span class="hljs-keyword">lazy</span> <span class="hljs-keyword">var</span> name <span class="hljs-operator">=</span> <span class="hljs-string">"thuyen"</span>
	<span class="hljs-keyword">var</span> alias: <span class="hljs-type">String</span> {
		<span class="hljs-keyword">mutating</span> <span class="hljs-keyword">get</span> { <span class="hljs-keyword">return</span> name }
	}
}
</code></pre>
<p>Another approach is to wrap lazy var inside a class <code>LazyBox</code>, like in <a href="https://oleb.net/blog/2015/12/lazy-properties-in-structs-swift">this article</a>. We will later use this approach as we can handle more issues such as concurrency.</p>
<pre><code class="hljs language-swift"><span class="hljs-keyword">final</span> <span class="hljs-keyword">class</span> <span class="hljs-title class_">LazyBox</span>&#x3C;<span class="hljs-type">T</span>> {
	<span class="hljs-keyword">private</span> <span class="hljs-keyword">var</span> _value: <span class="hljs-type">T</span>?
	<span class="hljs-keyword">private</span> <span class="hljs-keyword">let</span> compute: () -> <span class="hljs-type">T</span>
	<span class="hljs-keyword">init</span>(<span class="hljs-keyword">_</span> <span class="hljs-params">compute</span>: <span class="hljs-keyword">@escaping</span> () -> <span class="hljs-type">T</span>) {
		<span class="hljs-keyword">self</span>.compute <span class="hljs-operator">=</span> compute
	}

	<span class="hljs-keyword">var</span> value: <span class="hljs-type">T</span> {
		<span class="hljs-keyword">if</span> <span class="hljs-keyword">let</span> _value <span class="hljs-operator">=</span> _value { <span class="hljs-keyword">return</span> _value }
		<span class="hljs-keyword">let</span> v <span class="hljs-operator">=</span> compute()
		_value <span class="hljs-operator">=</span> v
		<span class="hljs-keyword">return</span> v
	}
}

<span class="hljs-keyword">struct</span> <span class="hljs-title class_">Person</span> {
	<span class="hljs-keyword">private</span> <span class="hljs-keyword">let</span> _name <span class="hljs-operator">=</span> <span class="hljs-type">LazyBox</span>&#x3C;<span class="hljs-type">String</span>> { <span class="hljs-string">"thuyen"</span> }
	<span class="hljs-keyword">var</span> alias: <span class="hljs-type">String</span> { <span class="hljs-keyword">return</span> _name.value }
}
</code></pre>
<h4 id="problem-2-concurrency">Problem 2: concurrency</h4>
<p>Another problem is that <code>lazy var</code> is not thread-safe which means the closure can get executed multiple times due to accesses from different threads. This is also mentioned in the Apple documentation:</p>
<blockquote>
<p>If a property marked with the <code>lazy</code> modifier is accessed by multiple threads simultaneously and the property has not yet been initialized, there is no guarantee that the property will be initialized only once.</p>
</blockquote>
<p>To prevent race condition, a simple implementation is to lock every read to the value:</p>
<pre><code class="hljs language-swift"><span class="hljs-keyword">final</span> <span class="hljs-keyword">class</span> <span class="hljs-title class_">LazyBox</span>&#x3C;<span class="hljs-type">T</span>> {
	<span class="hljs-keyword">private</span> <span class="hljs-keyword">let</span> lock: <span class="hljs-type">Synchronizing</span> <span class="hljs-operator">=</span> <span class="hljs-type">NSLock</span>()
	<span class="hljs-keyword">private</span> <span class="hljs-keyword">lazy</span> <span class="hljs-keyword">var</span> _lazyValue: <span class="hljs-type">T</span> <span class="hljs-operator">=</span> compute()
	<span class="hljs-operator">...</span>
	<span class="hljs-keyword">var</span> value: <span class="hljs-type">T</span> {
		<span class="hljs-keyword">return</span> lock.sync { _lazyValue }
	}
}
</code></pre>
<pre><code class="hljs language-swift"><span class="hljs-keyword">extension</span> <span class="hljs-title class_">Locking</span> {
	<span class="hljs-keyword">func</span> <span class="hljs-title function_">lock</span>()
	<span class="hljs-keyword">func</span> <span class="hljs-title function_">unlock</span>()
}

<span class="hljs-keyword">protocol</span> <span class="hljs-title class_">Synchronizing</span> {
	<span class="hljs-keyword">func</span> <span class="hljs-title function_">sync</span>&#x3C;<span class="hljs-type">T</span>>(<span class="hljs-params">execute</span>: () <span class="hljs-keyword">throws</span> -> <span class="hljs-type">T</span>) <span class="hljs-keyword">rethrows</span> -> <span class="hljs-type">T</span>
}

<span class="hljs-keyword">extension</span> <span class="hljs-title class_">Locking</span> {
	<span class="hljs-keyword">func</span> <span class="hljs-title function_">sync</span>&#x3C;<span class="hljs-type">T</span>>(<span class="hljs-params">execute</span>: () <span class="hljs-keyword">throws</span> -> <span class="hljs-type">T</span>) <span class="hljs-keyword">rethrows</span> -> <span class="hljs-type">T</span> {
		<span class="hljs-keyword">defer</span> { unlock() }
		lock()
		<span class="hljs-keyword">return</span> <span class="hljs-keyword">try</span> execute()
	}
}

<span class="hljs-comment">// Let's use NSLock for simplicity. Alternatives: DispatchQueue, pthread_mutex_t, semaphore...</span>
<span class="hljs-keyword">extension</span> <span class="hljs-title class_">NSLock</span>: <span class="hljs-title class_ inherited__">Locking</span>, <span class="hljs-title class_ inherited__">Synchronizing</span> { }
</code></pre>
<p>Now, it works as expected. However, the computation should only occur in the first read access. Locking every read will hurt performance a bit. Therefore, a better implementation is to lock the computation instead.</p>
<pre><code class="hljs language-swift"><span class="hljs-keyword">final</span> <span class="hljs-keyword">class</span> <span class="hljs-title class_">LazyBox</span>&#x3C;<span class="hljs-type">T</span>> {
	<span class="hljs-operator">...</span>
	<span class="hljs-keyword">var</span> value: <span class="hljs-type">T</span> {
		<span class="hljs-keyword">if</span> <span class="hljs-keyword">let</span> _value <span class="hljs-operator">=</span> _value { <span class="hljs-keyword">return</span> _value }
		<span class="hljs-keyword">return</span> lock.sync {
			<span class="hljs-comment">// Perform computation here</span>
			<span class="hljs-operator">...</span>
		}
	}
}
</code></pre>
<p>Note that performing computation more than once is still fine as long as they are synchronous and later computation reuses the result of previous ones.
We know for sure that in the next execution inside <code>lock.sync</code>, we definitely have <code>_value</code> computed. Then we can reuse that result, like this:</p>
<pre><code class="hljs language-swift"><span class="hljs-keyword">final</span> <span class="hljs-keyword">class</span> <span class="hljs-title class_">LazyBox</span>&#x3C;<span class="hljs-type">T</span>> {
	<span class="hljs-operator">...</span>
	<span class="hljs-keyword">var</span> value: <span class="hljs-type">T</span> {
		<span class="hljs-keyword">if</span> <span class="hljs-keyword">let</span> _value <span class="hljs-operator">=</span> _value { <span class="hljs-keyword">return</span> _value }
		<span class="hljs-keyword">return</span> lock.sync {
			<span class="hljs-comment">// Check again if the value is already computed (from the first one get called)</span>
			<span class="hljs-keyword">if</span> <span class="hljs-keyword">let</span> _value <span class="hljs-operator">=</span> _value { <span class="hljs-keyword">return</span> _value }
			<span class="hljs-keyword">let</span> v <span class="hljs-operator">=</span> compute()
			_value <span class="hljs-operator">=</span> v
			<span class="hljs-keyword">return</span> v
		}
	}
}
</code></pre>
<h3 id="benchmark">Benchmark</h3>
<p>I ran a performance test for the two implementations (locking computation only vs. locking every read).
The result shows a significant performance gain for the former one (0.002s as compared to 0.065s).
