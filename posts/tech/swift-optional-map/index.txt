13:T1bd3,<p>There are a number of cases in which we would like to initialize an object with
some parameters but what we currently have (i.e. these parameters),
unfortunately, are <em>Optional Type</em>. For instance, we have a <strong>urlString</strong> with
the type <strong>String?</strong> (or <em>Optional<String></em>), and we want to get a <strong>url</strong> from
<em>urlString</em>.</p>
<p>Basically, we expect to get a <em>url</em> of <strong>NSURL?</strong> (or <em>Optional<NSURL></em>). If
<strong>urlString == nil</strong>, the <em>url</em> should be <em>nil</em> as well.</p>
<p>However, the initialization of <strong>NSURL</strong> requires a <strong>string</strong> of <strong>String</strong>
(which is already unwrapped).</p>
<pre><code class="hljs language-swift"><span class="hljs-keyword">class</span> <span class="hljs-title class_">NSURL</span> {
   <span class="hljs-keyword">init?</span>(<span class="hljs-params">string</span> <span class="hljs-params">URLString</span>: <span class="hljs-type">String</span>)
}
</code></pre>
<p>Here comes a very popular way to deal with this <em>type-incompatibility</em>:</p>
<pre><code class="hljs language-swift"><span class="hljs-comment">// let urlString: String? = "www.myurl.com"</span>
<span class="hljs-keyword">if</span> <span class="hljs-keyword">let</span> urlString <span class="hljs-operator">=</span> urlString {
   <span class="hljs-keyword">let</span> url <span class="hljs-operator">=</span> <span class="hljs-type">NSURL</span>(string: urlString)
   <span class="hljs-comment">// Do stuff</span>
}
</code></pre>
<p>In case we wish to use the <em>url</em> outside the <em>if-let block</em>, it’s quite common
that we make <em>url</em> mutable and assign its value inside the condition block.</p>
<pre><code class="hljs language-swift"><span class="hljs-comment">// let urlString: String? = "www.myurl.com"</span>
<span class="hljs-keyword">var</span> url: <span class="hljs-type">NSURL</span>?
<span class="hljs-keyword">if</span> <span class="hljs-keyword">let</span> urlString <span class="hljs-operator">=</span> urlString {
   url <span class="hljs-operator">=</span> <span class="hljs-type">NSURL</span>(string: urlString)
}
</code></pre>
<p>This looks acceptable but:</p>
<ul>
<li><em>A few lines of code should have not been added for such a circumstance like this</em></li>
<li><em>The only reason to change url from immutable to mutable (from let … to var …)
is merely this type-incompatibility</em></li>
</ul>
<h4 id="map-is-coming-for-the-rescue"><em>Map</em> is coming for the rescue</h4>
<p>Luckily, we have a function <strong>map</strong> on <strong>Optional</strong>. (If Swift does not support,
we could define it :D). I am quite sure you also have used <strong>map</strong> on an
<strong>Array</strong> a lot, right?</p>
<p>For the sake of better understanding, I removed some unnecessary keywords that
are not directly relevant to this topic.</p>
<pre><code class="hljs language-swift"><span class="hljs-keyword">enum</span> <span class="hljs-title class_">Optional</span>&#x3C;<span class="hljs-type">Wrapped</span>> {
   map<span class="hljs-operator">&#x3C;</span><span class="hljs-type">U</span><span class="hljs-operator">></span>(f: (<span class="hljs-type">Wrapped</span>) -> <span class="hljs-type">U</span>) -> <span class="hljs-type">U</span>?
}
</code></pre>
<p>Now, we could init an object with a parameter of <em>Optional</em>, like this:</p>
<pre><code class="hljs language-vbnet">// <span class="hljs-keyword">let</span> urlString: <span class="hljs-type">String</span>? = <span class="hljs-string">"www.myurl.com"</span>
<span class="hljs-keyword">let</span> url = urlString.map { NSURL(<span class="hljs-type">string</span>: $<span class="hljs-number">0</span> }
</code></pre>
<p>Of course we could use this tip with other functions apart from initializations</p>
<pre><code class="hljs language-bash">// <span class="hljs-built_in">let</span> urlString: String? = <span class="hljs-string">"www.myurl.com"</span>
<span class="hljs-built_in">let</span> data = urlString.map { fetchDataFromPath(<span class="hljs-variable">$0</span> }   // data: NSData?
</code></pre>
<h4 id="how-about-a-function-with-2-parameters">How about a function with 2 parameters?</h4>
<pre><code class="hljs language-swift">fetchDataFromPath(path: <span class="hljs-type">String</span>, withToken: <span class="hljs-type">String</span>)
</code></pre>
<p>We can come up with the idea to turn a 2 optionals into an optional of
<strong>Tuple</strong>. Lets define an *operator *<strong>&#x3C;&#x26;></strong> to combine 2 optionals</p>
<pre><code class="hljs language-swift"><span class="hljs-keyword">infix</span> <span class="hljs-keyword">operator</span> <span class="hljs-title">&#x3C;&#x26;></span> { associativity left }

<span class="hljs-keyword">func</span> <span class="hljs-title function_">&#x3C;&#x26;>&#x3C;</span><span class="hljs-type">T</span>, <span class="hljs-type">U</span><span class="hljs-operator">></span> (left: <span class="hljs-type">Optional</span>&#x3C;<span class="hljs-type">T</span>>, right: <span class="hljs-type">Optional</span>&#x3C;<span class="hljs-type">U</span>>) -> <span class="hljs-type">Optional</span>&#x3C;(<span class="hljs-type">T</span>, <span class="hljs-type">U</span>)> {
   <span class="hljs-keyword">if</span> <span class="hljs-keyword">let</span> left <span class="hljs-operator">=</span> left, right <span class="hljs-operator">=</span> right {
      <span class="hljs-keyword">return</span> (left, right)
   }
   <span class="hljs-keyword">return</span> <span class="hljs-literal">nil</span>
}
</code></pre>
<p>Or we could define it in a more concise way using <em>flatMap</em> on <em>Optional</em>.</p>
<pre><code class="hljs language-swift"><span class="hljs-keyword">infix</span> <span class="hljs-keyword">operator</span> <span class="hljs-title">&#x3C;&#x26;></span> { associativity left }

<span class="hljs-keyword">func</span> <span class="hljs-title function_">&#x3C;&#x26;>&#x3C;</span><span class="hljs-type">T</span>, <span class="hljs-type">U</span><span class="hljs-operator">></span> (left: <span class="hljs-type">Optional</span>&#x3C;<span class="hljs-type">T</span>>, right: <span class="hljs-type">Optional</span>&#x3C;<span class="hljs-type">U</span>>) -> <span class="hljs-type">Optional</span>&#x3C;(<span class="hljs-type">T</span>, <span class="hljs-type">U</span>)> {
   <span class="hljs-keyword">return</span> left.flatMap { x <span class="hljs-keyword">in</span> right.flatMap { y <span class="hljs-keyword">in</span> (x, y) } }
}
</code></pre>

<span class="hljs-keyword">let</span> data <span class="hljs-operator">=</span> (urlString <span class="hljs-operator">&#x3C;&#x26;></span> token).map { fetchDataFromPath(<span class="hljs-variable">$0</span>, withToken: <span class="hljs-variable">$1</span>) }
</code></pre>
