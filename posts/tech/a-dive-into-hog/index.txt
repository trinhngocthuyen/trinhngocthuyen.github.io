13:Te047,<p>In this post, we will dive into Histogram of Oriented Gradients (HOG), a common technique used to extract features of images... And then implement it in python (in order to comprehend it).</p>
<h3 id="feature-descriptor">Feature Descriptor</h3>
<p>A <em>feature descriptor</em> is a representation of an image (or image patch) simplifying the image by extracting useful information and throwing away irrelevant information.</p>
<p>A <em>feature descriptor</em> converts an image of size <span class="katex"><span class="katex-mathml"><math xmlns="http://www.w3.org/1998/Math/MathML"><semantics><mrow><mi>W</mi><mo>×</mo><mi>H</mi><mo>×</mo><mn>3</mn></mrow><annotation encoding="application/x-tex">W \times H \times 3</annotation></semantics></math></span><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:0.7667em;vertical-align:-0.0833em;"></span><span class="mord mathnormal" style="margin-right:0.1389em;">W</span><span class="mspace" style="margin-right:0.2222em;"></span><span class="mbin">×</span><span class="mspace" style="margin-right:0.2222em;"></span></span><span class="base"><span class="strut" style="height:0.7667em;vertical-align:-0.0833em;"></span><span class="mord mathnormal" style="margin-right:0.0813em;">H</span><span class="mspace" style="margin-right:0.2222em;"></span><span class="mbin">×</span><span class="mspace" style="margin-right:0.2222em;"></span></span><span class="base"><span class="strut" style="height:0.6444em;"></span><span class="mord">3</span></span></span></span> to a feature vector of length <span class="katex"><span class="katex-mathml"><math xmlns="http://www.w3.org/1998/Math/MathML"><semantics><mrow><mi>N</mi></mrow><annotation encoding="application/x-tex">N</annotation></semantics></math></span><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:0.6833em;"></span><span class="mord mathnormal" style="margin-right:0.109em;">N</span></span></span></span></p>
<h4 id="what-is-considered-useful">What is considered <em>useful</em>?</h4>
<p>It depends on purpose. For example, it might not be good for image viewing, but for classification since using it produces good results.</p>
<p>In the HOG feature descriptor, the distribution (histograms) of directions of gradients (oriented gradients) are used as features. Gradients of an image are useful because the mangitude of gradients is large around edges and corners which contain a lot more information about object shape than flat regions.</p>

<p>However, I use the three approximations above because it is consistent with the implementation of <code>np.gradient()</code> which is a <em>standardly correct function</em> I can use to check my own code. Another reason is that <code>skimage.feature.hog</code> is implemented in the same way. I will be able to compare my result with that computed by <code>skimage</code>.</p>
<pre><code class="hljs language-python"><span class="hljs-keyword">def</span> <span class="hljs-title function_">compute_gradient</span>(<span class="hljs-params">image: np.ndarray</span>):
    <span class="hljs-string">"""
    Compute gradient of an image by rows and columns
    """</span>
    gx = np.zeros_like(image)
    gy = np.zeros_like(image)
    <span class="hljs-comment"># Central difference</span>
    gx[:, <span class="hljs-number">1</span>:-<span class="hljs-number">1</span>] = (image[:, <span class="hljs-number">2</span>:] - image[:, :-<span class="hljs-number">2</span>]) / <span class="hljs-number">2</span>
    gy[<span class="hljs-number">1</span>:-<span class="hljs-number">1</span>, :] = (image[<span class="hljs-number">2</span>:, :] - image[:-<span class="hljs-number">2</span>, :]) / <span class="hljs-number">2</span>

    <span class="hljs-comment"># Forward difference</span>
    gx[:, <span class="hljs-number">0</span>] = image[:, <span class="hljs-number">1</span>] - image[:, <span class="hljs-number">0</span>]
    gy[<span class="hljs-number">0</span>, :] = image[<span class="hljs-number">1</span>, :] - image[<span class="hljs-number">0</span>, :]

    <span class="hljs-comment"># Backward difference</span>
    gx[:, -<span class="hljs-number">1</span>] = image[:, -<span class="hljs-number">1</span>] - image[:, -<span class="hljs-number">2</span>]
    gy[-<span class="hljs-number">1</span>, :] = image[-<span class="hljs-number">1</span>, :] - image[-<span class="hljs-number">2</span>, :]

    <span class="hljs-keyword">return</span> gx, gy

<pre><code class="hljs language-python"><span class="hljs-keyword">def</span> <span class="hljs-title function_">compute_hog_cell</span>(<span class="hljs-params">n_orientations: <span class="hljs-built_in">int</span>, magnitudes: np.ndarray, orientations: np.ndarray</span>) -> np.ndarray:
    <span class="hljs-string">"""
    Compute 1 HOG feature of a cell. Return a row vector of size `n_orientations`
    """</span>
    bin_width = <span class="hljs-built_in">int</span>(<span class="hljs-number">180</span> / n_orientations)
    hog = np.zeros(n_orientations)
    <span class="hljs-keyword">for</span> i <span class="hljs-keyword">in</span> <span class="hljs-built_in">range</span>(orientations.shape[<span class="hljs-number">0</span>]):
        <span class="hljs-keyword">for</span> j <span class="hljs-keyword">in</span> <span class="hljs-built_in">range</span>(orientations.shape[<span class="hljs-number">1</span>]):
            orientation = orientations[i, j]
            lower_bin_idx = <span class="hljs-built_in">int</span>(orientation / bin_width)
            hog[lower_bin_idx] += magnitudes[i, j]

    <span class="hljs-keyword">return</span> hog / (magnitudes.shape[<span class="hljs-number">0</span>] * magnitudes.shape[<span class="hljs-number">1</span>])

<h4 id="step-4-block-normalization">Step 4: Block normalization</h4>
<p>Up to this step, we have created a histogram based on the gradients of the image. However, the gradients are sensitive to illumination. For example, if we darken the image by a half of light, the gradient magnitudes will decrease twice which means the histogram values change by half. A practical way to alleviate this dependence is normalizing the histogram.</p>
<p>We could perform normalization in larger regions. This stage is called <em><strong>block normalization</strong></em>. For example, we could take a block of 4 (square aligned) cells to normalize and take the result as a final histogram feature. We slide to the right and then down to get histogram features for other blocks.</p>
<p>📝 Note:</p>
<ul>
<li>These blocks could overlap each other.</li>
<li>Each histogram of a block is a row vector of size 36 (4 cells/block)</li>
</ul>
<p>In this examples, with 4 cells/block, there are <span class="katex"><span class="katex-mathml"><math xmlns="http://www.w3.org/1998/Math/MathML"><semantics><mrow><mn>63</mn><mo>×</mo><mn>63</mn></mrow><annotation encoding="application/x-tex">63 \times 63</annotation></semantics></math></span><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:0.7278em;vertical-align:-0.0833em;"></span><span class="mord">63</span><span class="mspace" style="margin-right:0.2222em;"></span><span class="mbin">×</span><span class="mspace" style="margin-right:0.2222em;"></span></span><span class="base"><span class="strut" style="height:0.6444em;"></span><span class="mord">63</span></span></span></span> blocks. So, we <span class="katex"><span class="katex-mathml"><math xmlns="http://www.w3.org/1998/Math/MathML"><semantics><mrow><mn>63</mn><mo>×</mo><mn>63</mn></mrow><annotation encoding="application/x-tex">63 \times 63</annotation></semantics></math></span><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:0.7278em;vertical-align:-0.0833em;"></span><span class="mord">63</span><span class="mspace" style="margin-right:0.2222em;"></span><span class="mbin">×</span><span class="mspace" style="margin-right:0.2222em;"></span></span><span class="base"><span class="strut" style="height:0.6444em;"></span><span class="mord">63</span></span></span></span> histogram vectors of size 36.</p>
<p><span class="katex"><span class="katex-mathml"><math xmlns="http://www.w3.org/1998/Math/MathML"><semantics><mrow><mtext>  </mtext><mo>⟹</mo><mtext>  </mtext><mn>63</mn><mo>×</mo><mn>63</mn><mo>×</mo><mn>36</mn><mo>=</mo><mn>142884</mn></mrow><annotation encoding="application/x-tex">\implies 63 \times 63 \times 36 = 142884</annotation></semantics></math></span><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:0.549em;vertical-align:-0.024em;"></span><span class="mspace" style="margin-right:0.2778em;"></span><span class="mrel">⟹</span><span class="mspace" style="margin-right:0.2778em;"></span><span class="mspace" style="margin-right:0.2778em;"></span></span><span class="base"><span class="strut" style="height:0.7278em;vertical-align:-0.0833em;"></span><span class="mord">63</span><span class="mspace" style="margin-right:0.2222em;"></span><span class="mbin">×</span><span class="mspace" style="margin-right:0.2222em;"></span></span><span class="base"><span class="strut" style="height:0.7278em;vertical-align:-0.0833em;"></span><span class="mord">63</span><span class="mspace" style="margin-right:0.2222em;"></span><span class="mbin">×</span><span class="mspace" style="margin-right:0.2222em;"></span></span><span class="base"><span class="strut" style="height:0.6444em;"></span><span class="mord">36</span><span class="mspace" style="margin-right:0.2778em;"></span><span class="mrel">=</span><span class="mspace" style="margin-right:0.2778em;"></span></span><span class="base"><span class="strut" style="height:0.6444em;"></span><span class="mord">142884</span></span></span></span> numbers</p>
<pre><code class="hljs language-python"><span class="hljs-keyword">def</span> <span class="hljs-title function_">normalize_vector</span>(<span class="hljs-params">v, eps=<span class="hljs-number">1e-5</span></span>):
    <span class="hljs-string">"""
    Return a normalized vector (which has norm2 as 1) 
    """</span>
    <span class="hljs-comment"># eps is used to prevent zero divide exceptions (in case v is zero)</span>
    <span class="hljs-keyword">return</span> v / np.sqrt(np.<span class="hljs-built_in">sum</span>(v ** <span class="hljs-number">2</span>) + eps ** <span class="hljs-number">2</span>) 


<span class="hljs-keyword">def</span> <span class="hljs-title function_">compute_hog_features</span>(<span class="hljs-params">image: np.ndarray,
                         n_orientations: <span class="hljs-built_in">int</span> = <span class="hljs-number">9</span>, pixels_per_cell: (<span class="hljs-params"><span class="hljs-built_in">int</span>, <span class="hljs-built_in">int</span></span>) = (<span class="hljs-params"><span class="hljs-number">8</span>, <span class="hljs-number">8</span></span>),
                         cells_per_block: (<span class="hljs-params"><span class="hljs-built_in">int</span>, <span class="hljs-built_in">int</span></span>) = (<span class="hljs-params"><span class="hljs-number">1</span>, <span class="hljs-number">1</span></span>)</span>) -> np.ndarray:
    <span class="hljs-string">"""
    Compute HOG features of an image. Return a row vector
    """</span>
    gx, gy = compute_gradient(image)
    sy, sx = gx.shape
    cx, cy = pixels_per_cell
    bx, by = cells_per_block

    magnitudes = np.hypot(gx, gy)   <span class="hljs-comment"># = np.sqrt(gx**2 + gy**2)</span>
    orientations = np.rad2deg(np.arctan2(gy, gx)) % <span class="hljs-number">180</span>

    n_cellsx = <span class="hljs-built_in">int</span>(sx / cx) <span class="hljs-comment"># Number of cells in x axis</span>
    n_cellsy = <span class="hljs-built_in">int</span>(sy / cy) <span class="hljs-comment"># Number of cells in y axis</span>
    n_blocksx = <span class="hljs-built_in">int</span>(n_cellsx - bx) + <span class="hljs-number">1</span>
    n_blocksy = <span class="hljs-built_in">int</span>(n_cellsy - by) + <span class="hljs-number">1</span>

    hog_cells = np.zeros((n_cellsx, n_cellsy, n_orientations))

    prev_x = <span class="hljs-number">0</span>
    <span class="hljs-comment"># Compute HOG of each cell</span>
    <span class="hljs-keyword">for</span> it_x <span class="hljs-keyword">in</span> <span class="hljs-built_in">range</span>(n_cellsx):
        prev_y = <span class="hljs-number">0</span>
        <span class="hljs-keyword">for</span> it_y <span class="hljs-keyword">in</span> <span class="hljs-built_in">range</span>(n_cellsy):
            magnitudes_patch = magnitudes[prev_y:prev_y + cy, prev_x:prev_x + cx]
            orientations_patch = orientations[prev_y:prev_y + cy, prev_x:prev_x + cx]

            hog_cells[it_y, it_x] = compute_hog_cell(n_orientations, magnitudes_patch, orientations_patch)

            prev_y += cy
        prev_x += cx

    hog_blocks_normalized = np.zeros((n_blocksx, n_blocksy, n_orientations))

    <span class="hljs-comment"># Normalize HOG by block</span>
    <span class="hljs-keyword">for</span> it_blocksx <span class="hljs-keyword">in</span> <span class="hljs-built_in">range</span>(n_blocksx):
        <span class="hljs-keyword">for</span> it_blocky <span class="hljs-keyword">in</span> <span class="hljs-built_in">range</span>(n_blocksy):
            hog_block = hog_cells[it_blocky:it_blocky + by, it_blocksx:it_blocksx + bx].ravel()
            hog_blocks_normalized[it_blocky, it_blocksx] = normalize_vector(hog_block)

    <span class="hljs-keyword">return</span> hog_blocks_normalized.ravel()
</code></pre>
