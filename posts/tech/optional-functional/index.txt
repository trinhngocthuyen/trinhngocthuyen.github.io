13:T1700,<p>I have just read the chapter 5 of the book <a href="http://www.objc.io/books/fpinswift/">"Functional programming in
Swift"</a>. This chapter is about
<strong>"Optionals"</strong>. And there are some interesting things I want to share.</p>
<p>First of all, one of the most frustrating problem that developers have been
experiencing is working with <code>nil</code> (or <code>null</code>). Intuitively, <code>nil</code> (or <code>null</code>) implicitly means
<strong>"failure"</strong>. For instance, you look up in a dictionary but cannot find it,
then you got <code>nil</code>:</p>
<pre><code class="hljs language-swift"><span class="hljs-keyword">let</span> mutualFriends<span class="hljs-operator">=</span> [
    <span class="hljs-string">"Anna"</span>: <span class="hljs-number">137</span>,
    <span class="hljs-string">"Chris"</span>: <span class="hljs-number">222</span>,
    <span class="hljs-string">"Dan"</span>: <span class="hljs-number">80</span>,
    <span class="hljs-string">"Fred"</span>: <span class="hljs-number">152</span>]   <span class="hljs-comment">// mutualFriends: &#x3C;String, Int?></span>
<span class="hljs-keyword">let</span> x <span class="hljs-operator">=</span> mutualFriends[<span class="hljs-string">"John"</span>]      <span class="hljs-comment">// x = nil</span>
</code></pre>
<p>As usual, we want to assign x a <em>default value</em> instead of <code>nil</code>. We can customize
it in this way:</p>
<pre><code class="hljs language-swift"><span class="hljs-keyword">var</span> x: <span class="hljs-type">Int</span> <span class="hljs-operator">=</span> defaultValue
<span class="hljs-keyword">if</span> <span class="hljs-keyword">let</span> y<span class="hljs-operator">=</span> mutualFriends[<span class="hljs-string">"John"</span>] {
    x <span class="hljs-operator">=</span> y
}
</code></pre>
<p>In fact, Swift supports an operator <code>??</code> to make this sort of declaration
more concise:</p>
<pre><code class="hljs language-swift"><span class="hljs-keyword">let</span> x <span class="hljs-operator">=</span> mutualFriends[<span class="hljs-string">"John"</span>] <span class="hljs-operator">??</span> defaultValue
</code></pre>
<p>This means: If the left of <code>??</code> is <code>nil</code>, then return the value of the right.
Let’s dive a little bit into the operator <code>??</code>. Roughly speaking it can be
defined as follows:</p>
<pre><code class="hljs language-swift"><span class="hljs-comment">// Custom ?? function</span>
<span class="hljs-keyword">func</span> <span class="hljs-title function_">??&#x3C;</span><span class="hljs-type">T</span><span class="hljs-operator">></span>(optional: <span class="hljs-type">T</span>?, defaultValue: <span class="hljs-type">T</span>) -> <span class="hljs-type">T</span> {
    <span class="hljs-keyword">if</span> <span class="hljs-keyword">let</span> x <span class="hljs-operator">=</span> <span class="hljs-keyword">optional</span> {
        <span class="hljs-keyword">return</span> x
    } <span class="hljs-keyword">else</span> {
        <span class="hljs-keyword">return</span> defaultValue
    }
}
</code></pre>
<p>This function works perfectly despite one problem: <code>defaultValue</code> is always
computed even if the optional is not <code>nil</code>. Generally, the reason is that it is
computed before passing in the function parameters. What we desire is that the
computation of the defaultValue is only executed inside the else branch.</p>
<p>We can resolve it by passing <strong>"HOW" defaultValue is computed</strong> instead of the
value computed already. Intuitively, it reminds us of using a <strong>function</strong> as a
parameter. And see what it leads to:</p>
<pre><code class="hljs language-swift"><span class="hljs-comment">// Custom ?? function</span>
<span class="hljs-keyword">func</span> <span class="hljs-title function_">??&#x3C;</span><span class="hljs-type">T</span><span class="hljs-operator">></span>(optional: <span class="hljs-type">T</span>?, computeDefaultValue: () -> <span class="hljs-type">T</span>) -> <span class="hljs-type">T</span> {
    <span class="hljs-keyword">if</span> <span class="hljs-keyword">let</span> x <span class="hljs-operator">=</span> <span class="hljs-keyword">optional</span> {
        <span class="hljs-keyword">return</span> x
    } <span class="hljs-keyword">else</span> {
        <span class="hljs-keyword">return</span> computeDefaultValue()
}
</code></pre>
<p>Come back to our example, we can use the operator <code>??</code> like this:</p>
<pre><code class="hljs language-swift"><span class="hljs-keyword">let</span> x <span class="hljs-operator">=</span> mutualFriends[<span class="hljs-string">"John"</span>] <span class="hljs-operator">??</span> { myDefaultValue }
</code></pre>
<p>A little ugly with the closure, right? Fortunately, Swift does support
<em>"autoclosure type attribute"</em> to deal with with. Let’s look at the official
function of <code>??</code>:</p>
<pre><code class="hljs language-swift"><span class="hljs-comment">// Swift 1.2 ?? function</span>
<span class="hljs-keyword">func</span> <span class="hljs-title function_">??&#x3C;</span><span class="hljs-type">T</span><span class="hljs-operator">></span>(optional: <span class="hljs-type">T</span>?, <span class="hljs-keyword">@autoclosure</span> defaultValue: () -> <span class="hljs-type">T</span>) -> <span class="hljs-type">T</span> {
    <span class="hljs-operator">......</span>
}
<span class="hljs-keyword">let</span> x <span class="hljs-operator">=</span> mutualFriends[<span class="hljs-string">"John"</span>] <span class="hljs-operator">??</span> myDefaultValue
</code></pre>
<p><img src="/images/ios/operator_or_autoclosure.png" alt=""></p>

<code>myDefaultValue</code> to a corresponding function. Everything seems quite natural so
