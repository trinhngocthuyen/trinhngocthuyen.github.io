13:T20e4,<p>When it comes to iOS architecture, MVVM is one of the most favorable candidates. Not only does it provide higher testability than MVC but also this architecture is lightweight as compared to its counterparts such as VIPER. Despite that, proper approaches should be adopted to take advantage of MVVM. Otherwise, we could end up with an alike version of MVC with an additional component (ViewModel).</p>
<p>This post introduces an approach call <em>Inputs - Outputs</em>, currently used at Kickstarter. You can see a high fraction of this style in the Kickstarter iOS app.</p>
<p><em>Disclaimer: This approach is nothing but a convention. Don't get confused it with an architecture.</em></p>
<h3 id="principles">Principles</h3>
<ul>
<li><code>inputs</code> is a set of actions and events that have impacts on viewModel such as the tap action on a button, or the viewDidLoad event.</li>
<li><code>outputs</code> represents changes that views should reflect.</li>
<li>Since <code>ouputs</code> may change over time, it's best to return an <code>Observable</code> (in RxSwift context) for each ouput.</li>
<li>Behaviors defined in <code>inputs</code> should not be expressed as <code>Variable</code> because we don't need the inputs to be obseravable.</li>
</ul>

<h3 id="why-inputs---outputs">Why Inputs - Outputs?</h3>
<p>First of all, by using protocols like this, we achieve higher level of abstraction. Therefore, our code is more behavior-oriented and easier to test.</p>
<p>Another advantage of this protocol-based convention is readability in unit tests. Let's take a look at the two simple tests below:</p>
<pre><code class="hljs language-swift"><span class="hljs-keyword">func</span> <span class="hljs-title function_">test_When_PasswordIsEmpty_Then_InputIsInvalid</span>() {
	viewModel.inputs.viewDidLoad()
	viewModel.inputs.type(email: <span class="hljs-string">"abc@xyz.com"</span>)

	<span class="hljs-comment">// `grabLatestValue` is just an utility function we can write to retrieve</span>
	<span class="hljs-comment">// the latest value in the stream (during a specific period of time).</span>
	<span class="hljs-comment">// `RxBlocking` comes for the rescue.</span>
	<span class="hljs-keyword">let</span> validInput <span class="hljs-operator">=</span> grabLatestValue(viewModel.outputs.validInput, duration: <span class="hljs-number">1</span>)
	expect(validInput).to(beFalse())
}

<span class="hljs-keyword">func</span> <span class="hljs-title function_">test_When_Submitting_Then_ShouldShowLoadingAndThenHideWhenCompleted</span>() {
	viewModel.inputs.viewDidLoad()
	viewModel.inputs.type(email: <span class="hljs-string">"abc@xyz.com"</span>)
	viewModel.inputs.type(password: <span class="hljs-string">"Password0"</span>)
	viewModel.inputs.tapSubmit()

	<span class="hljs-keyword">let</span> loadingStates <span class="hljs-operator">=</span> grabLatestValue(viewModel.outputs.isLoading, duration: <span class="hljs-number">1</span>)
	expect(loadingStates).to(beEqual([<span class="hljs-literal">true</span>, <span class="hljs-literal">false</span>]))
}
</code></pre>
<p>By looking at the codes related to <code>inputs</code> calls, we quickly have a sense of the scenarios we are trying to simulate. Similarly, what we expect to see are reflected upon <code>outputs</code>.</p>
