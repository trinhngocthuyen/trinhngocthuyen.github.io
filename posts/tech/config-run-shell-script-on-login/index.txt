13:T114b,<p>Sometimes, I got frustrated with the fact that my SSH keys were cleared from agents after rebooting. As a result, I had to manually add those keys to agents every time my computer restarted (although it's not occasional)... It's best to automate such tasks - I thought. So, I asked <em>"how to run a shell script on login?"</em>. And here are some solutions.</p>
<h3 id="option-1-automator">Option 1: Automator</h3>
<ul>
<li>Open <em><strong>Automator</strong></em>. Choose <em><strong>Application</strong></em> from the templates.</li>
<li>Choose <em><strong>Run Shell Script</strong></em> from the action list.</li>
<li>Paste your code there (see the figure below). For convenience, I launch another script which I could easily open and edit.
<img src="/images/misc/shell_script_on_login_1.png" style="width:600px">
</li>
<li><em><strong>Save</strong></em> the application.</li>
<li>Open <em><strong>System Preferences</strong></em> -> <em><strong>Users &#x26; Groups</strong></em> -> <em><strong>Login Items</strong></em>. Then, add the application you have just created.</li>
</ul>
<h3 id="option-2-daemons">Option 2: Daemons</h3>
<ul>
<li>Create a plist file in <code>~/Library/LaunchAgents</code> (ex. <code>com.thuyen.loginscript.plist</code>):</li>
</ul>

<p>🔖 <em><u>Note</u></em>: For some reasons, the plist does not work with the executable path like <code>~/scripts/on_login.sh</code> (<code>~</code> for HOME directory). Explicit path <code>/Users/thuyentrinh/scripts/on_login.sh</code> solves the problems... My MacOS is 10.13; maybe that matters 🙄. I did not test it in earlier MacOS versions.</p>
<ul>
<li>Load the daemon:</li>
</ul>
<pre><code class="hljs language-bash">launchctl load ~/Library/LaunchAgents/com.thuyen.loginscript.plist
</code></pre>
<ul>
<li>You could verify if the daemon is loaded by listing out services</li>
</ul>
<pre><code class="hljs language-perl">launchctl list | <span class="hljs-keyword">grep</span> loginscript
</code></pre>
