13:T14b5,<p>Everytime we create a new file, Xcode suggests some built-in templates that may be useful. Luckily, we could also create our own templates.</p>
<p>Technically, Xcode looks for templates in <code>~/Library/Developer/Xcode/Templates</code> and <code>/Applications/Xcode.app/Contents/Developer/Library/Xcode/Templates</code>. Let's call the first path P1 and the second path P2. A template could be iferred by a folder <code>MY_TEMPLATE.xctemplate</code>.</p>
<p>Assume we gonna create a template for MVVM. It should create 3 files: <code>[Feature]Model.swift</code>, <code>[Feature]ViewModel.swift</code> and <code>[Feature]ViewController.swift</code>. Ex: <code>LoginModel.swift</code>, <code>LoginViewModel.swift</code> and <code>LoginViewController.swift</code></p>
<ul>
<li>First of all, create a xctemplate folder in P1: <code>~/Library/Developer/Xcode/Templates/Custom/MVVM.xctemplate</code>. Note: we should place the xctemplate folder in P1 instead of P2 b/c P2 is only particular for a specific version of Xcode.</li>
<li>The folder should include these files (you could copy it from P2 and modify it a little bit):
<ul>
<li><code>___FILEBASENAME___Model.swift</code></li>
<li><code>___FILEBASENAME___ViewModel.swift</code></li>
<li><code>___FILEBASENAME___ViewController.swift</code></li>
</ul>
</li>
<li>Create placeholder for the 3 files above. Here is the sample for <code>___FILEBASENAME__ViewController.swift</code>:</li>
</ul>
