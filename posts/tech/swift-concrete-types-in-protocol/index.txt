13:T113a,<p>Suppose you are writing a language-translation app. The requirement is quite
simple: <em>Given a word, translate it from a language into another language.</em></p>
<p>Here’s a simple approach. We define a protocol <code>TranslatorType</code> as follows</p>
<pre><code class="hljs language-swift"><span class="hljs-keyword">class</span> <span class="hljs-title class_">Language</span> {
   <span class="hljs-keyword">var</span> words: [<span class="hljs-type">Word</span>] <span class="hljs-operator">=</span> []
}
<span class="hljs-keyword">class</span> <span class="hljs-title class_">VI_Language</span>: <span class="hljs-title class_ inherited__">Language</span> { }
<span class="hljs-keyword">class</span> <span class="hljs-title class_">EN_Language</span>: <span class="hljs-title class_ inherited__">Language</span> { }
<span class="hljs-keyword">protocol</span> <span class="hljs-title class_">TranslatorType</span> {
   <span class="hljs-keyword">var</span> sourceLanguage: <span class="hljs-type">Language</span> { <span class="hljs-keyword">get</span> }
   <span class="hljs-keyword">var</span> destLanguage: <span class="hljs-type">Language</span> { <span class="hljs-keyword">get</span> }
   <span class="hljs-keyword">func</span> <span class="hljs-title function_">translate</span>(<span class="hljs-params">word</span>: <span class="hljs-type">Word</span>) -> <span class="hljs-type">Word</span>
}
</code></pre>
<p>The <code>VI_EN_Translator</code> (which translates Vietnamese into English) should
conform to that protocol with <code>sourceLanguage</code> as <code>VI_Language</code> and
<code>destLanguage</code> as <code>EN_Language</code>.</p>
<pre><code class="hljs language-swift"><span class="hljs-keyword">class</span> <span class="hljs-title class_">VI_EN_Translator</span>: <span class="hljs-title class_ inherited__">TranslatorType</span> {
   <span class="hljs-keyword">private(set)</span> <span class="hljs-keyword">var</span> sourceLanguage <span class="hljs-operator">=</span> <span class="hljs-type">VI_Language</span>()
   <span class="hljs-keyword">private(set)</span> <span class="hljs-keyword">var</span> destLanguage <span class="hljs-operator">=</span> <span class="hljs-type">EN_Language</span>()
   <span class="hljs-keyword">func</span> <span class="hljs-title function_">translate</span>(<span class="hljs-params">word</span>: <span class="hljs-type">Word</span>) -> <span class="hljs-type">Word</span> {
      <span class="hljs-operator">...</span>
   }
}
</code></pre>
<p>It seems reasonable. Unfortunately, I got an error from XCode: <em>“Type
VI_EN_Translator does not conform to protocol TranslatorType”</em> despite the fact
that <code>VI_Language</code> is a subclass of <code>Language</code>. I don’t want to change
sourceLanguage to Language and cast it to <code>VI_Language</code> every I use it.</p>
<p><img src="/images/ios/concrete_type_protocol_error.png" alt=""></p>
<p>Not sure wheter we should blame Swift/XCode/Apple for that. Here’s a tip to
overcome this issue:</p>
<pre><code class="hljs language-swift"><span class="hljs-keyword">class</span> <span class="hljs-title class_">VI_EN_Translator</span>: <span class="hljs-title class_ inherited__">TranslatorType</span> {
   <span class="hljs-keyword">private</span> <span class="hljs-keyword">var</span> _sourceLanguage <span class="hljs-operator">=</span> <span class="hljs-type">VI_Language</span>()
   <span class="hljs-keyword">private</span> <span class="hljs-keyword">var</span> _destLanguage <span class="hljs-operator">=</span> <span class="hljs-type">EN_Language</span>()
   <span class="hljs-keyword">var</span> sourceLanguage: <span class="hljs-type">Language</span> {
      <span class="hljs-keyword">return</span> _sourceLanguage
   }
   <span class="hljs-keyword">var</span> destLanguage: <span class="hljs-type">Language</span> {
      <span class="hljs-keyword">return</span> _destLanguage
   }
   <span class="hljs-keyword">func</span> <span class="hljs-title function_">translate</span>(<span class="hljs-params">word</span>: <span class="hljs-type">Word</span>) -> <span class="hljs-type">Word</span> {
      <span class="hljs-operator">...</span>
   }
}
</code></pre>
