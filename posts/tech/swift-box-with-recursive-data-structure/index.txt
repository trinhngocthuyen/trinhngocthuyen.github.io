<p>Despite that value types in general (and <em>enum/struct</em> in particular) bring a
lot of advantages, there are several limitations remaining. In particular:</p>
<ul>
<li>Both enum and struct do not support <strong>recursive</strong> data structure</li>
<li>Enum with a <strong>type-parameterized</strong> case is not allowed</li>
</ul>
<p>And Box is a micro framework to deal with the painful facts above.</p>
<h3 id="why">Why?</h3>
<p>First of all, lets find out the reasons for fact 1 and fact 2. We shall begin
with an example: <em>implementing a very familiar data struct: <code>LIST</code></em>.</p>
<p>A list could consist of a head and a tail, which is also a list. A list could be
nothing as well.</p>
<p><img src="/images/ios/list_recursive_ds.png" alt=""></p>
<p>Unfortunately, this code throws a compiler error. In fact, when XCode allocates
memory for <code>List&#x3C;Int></code> — for example, it couldn’t estimate how much is enough
for <code>Cons(Int, List&#x3C;Int>)</code> because it does not yet figure out the memory
capacity for <code>List&#x3C;Int></code> in <code>Cons(Int, List&#x3C;Int>)</code>.</p>
<p>Secondly, it does not accept a type-parameterized associated value: <code>Cons(T, List&#x3C;T>)</code>. ← For this, I still don’t know why.</p>
<h3 id="how-to-overcome">How to overcome?</h3>
<p>Luckily, <code>Box</code> is coming for the rescue. The idea is very simple: <em>make it
non-recursive, non-type-paramaterized</em> by using another data structure.
The new data structure’s responsibility is wrapping the value in a box. And when
you need the value, just unwrap the box.</p>
<pre><code class="hljs language-swift"><span class="hljs-keyword">class</span> <span class="hljs-title class_">Box</span>&#x3C;<span class="hljs-type">T</span>> {
    <span class="hljs-keyword">var</span> value: <span class="hljs-type">T</span>
    <span class="hljs-keyword">init</span>(<span class="hljs-keyword">_</span> <span class="hljs-params">value</span>: <span class="hljs-type">T</span>) {
        <span class="hljs-keyword">self</span>.value <span class="hljs-operator">=</span> value
    }
}

<span class="hljs-keyword">enum</span> <span class="hljs-title class_">List</span>&#x3C;<span class="hljs-type">T</span>> {
    <span class="hljs-keyword">case</span> <span class="hljs-type">Cons</span>(<span class="hljs-type">Box</span>&#x3C;<span class="hljs-type">T</span>>, <span class="hljs-type">Box</span>&#x3C;<span class="hljs-type">List</span>&#x3C;<span class="hljs-type">T</span>>>)
    <span class="hljs-keyword">case</span> <span class="hljs-type">Nil</span>

    <span class="hljs-keyword">init</span>(<span class="hljs-keyword">_</span> <span class="hljs-params">head</span>: <span class="hljs-type">T</span>, <span class="hljs-keyword">_</span> <span class="hljs-params">tail</span>: <span class="hljs-type">List</span>&#x3C;<span class="hljs-type">T</span>>) {
        <span class="hljs-keyword">self</span> <span class="hljs-operator">=</span> <span class="hljs-type">Cons</span>(<span class="hljs-type">Box</span>(head), <span class="hljs-type">Box</span>(tail))
    }
}
</code></pre>
<p>By this, <code>List&#x3C;T></code> is not recursive anymore. <em>But of course, it’s still
logically a recursion :D</em>. The compiler won’t complain about the memory
allocation problem because it can estimate how much memory a box takes.</p>
<h3 id="other-examples">Other examples</h3>
<p>If you use <code>ReactiveCocoa 3.0</code>, you will see <code>Box</code> as a submodule
of it. In fact, <code>RAC 3.0</code> includes another submodule called <code>Result</code>. This
micro framework also use <code>Box</code>, too.</p>
