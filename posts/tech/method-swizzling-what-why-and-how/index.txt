13:T20c5,<h3 id="what-is-method-swizzling">What is Method swizzling?</h3>
<p><em>Method swizzling</em> is a very powerful technique that takes advantage of dynamism. The core idea of this technique is to replace the real implementation of a method at runtime. With this power, we could be able to do a lot of cool stuffs.</p>
<p>Actually, this special feature is offered by the Objective-C runtime, via message dispatch. You could read my <a href="/tech/method-dispatch-in-swift">previous post</a> to have a clear picture of method dispatch in Swift.</p>
<h3 id="why-does-it-matter">Why does it matter?</h3>
<p>A very common case study of this is integrating analytics in your app. Take Google Analytics (GA) for example, each time user enters a screen, the app should call the GA APIs for page views tracking.</p>
<p>One could implement it simply by making GA requests once the method <code>viewWillAppear</code> of each view controllers is invoked.</p>
<p>However, there could be up to hundreds of view controllers in the app. Manually calling GA APIs in each controller is apparently ineffective although it only costs just a single line of code. Another drawback is that it is hard to control which one is missing. Also, you have limited ability to hook into the code of 3rd party libraries if necessary.</p>
<p>The problem appears to be quite simple with <em>method swizzling</em>. All you have to do is to write a custom function <code>_tracked_viewWillAppear</code> then swap it with the original function <code>viewWillAppear</code>. I will talk in detail later.</p>
<h3 id="how-to-swizzle-a-method">How to swizzle a method?</h3>
<p>The magic function you need to remember is <a href="https://developer.apple.com/documentation/objectivec/1418769-method_exchangeimplementations"><code>method_exchangeImplementations</code></a>:</p>
<pre><code class="hljs language-swift"><span class="hljs-keyword">func</span> <span class="hljs-title function_">method_exchangeImplementations</span>(<span class="hljs-keyword">_</span> <span class="hljs-params">m1</span>: <span class="hljs-type">Method</span>, <span class="hljs-keyword">_</span> <span class="hljs-params">m2</span>: <span class="hljs-type">Method</span>)
</code></pre>
<p>As the name reflects, the implementations of <code>m1</code> and <code>m2</code> get swapped after calling this function (if the exchange is successful). It means that an invocation to <code>m1</code> actually executes the code inside <code>m2</code> and vice versa.</p>
<pre><code class="hljs language-swift"><span class="hljs-keyword">let</span> selector1 <span class="hljs-operator">=</span> <span class="hljs-keyword">#selector</span>(<span class="hljs-type">UIViewController</span>.viewWillAppear(<span class="hljs-keyword">_</span>:))
<span class="hljs-keyword">let</span> selector2 <span class="hljs-operator">=</span> <span class="hljs-keyword">#selector</span>(<span class="hljs-type">UIViewController</span>._swizzled_viewWillAppear(<span class="hljs-keyword">_</span>:))
<span class="hljs-keyword">let</span> method1 <span class="hljs-operator">=</span> class_getInstanceMethod(<span class="hljs-type">UIViewController</span>.<span class="hljs-keyword">self</span>, selector1)<span class="hljs-operator">!</span>
<span class="hljs-keyword">let</span> method2 <span class="hljs-operator">=</span> class_getInstanceMethod(<span class="hljs-type">UIViewController</span>.<span class="hljs-keyword">self</span>, selector2)<span class="hljs-operator">!</span>
method_exchangeImplementations(method1, method2)
</code></pre>
<p>This is what the function <code>_swizzled_viewWillAppear</code> looks like:</p>
<pre><code class="hljs language-swift"><span class="hljs-keyword">extension</span> <span class="hljs-title class_">UIViewController</span> {
    <span class="hljs-keyword">@objc</span> <span class="hljs-keyword">dynamic</span> <span class="hljs-keyword">func</span> <span class="hljs-title function_">_swizzled_viewWillAppear</span>(<span class="hljs-keyword">_</span> <span class="hljs-params">animated</span>: <span class="hljs-type">Bool</span>) {
        <span class="hljs-type">NSLog</span>(<span class="hljs-string">"Enter screen: <span class="hljs-subst">\(<span class="hljs-built_in">type</span>(of: <span class="hljs-keyword">self</span>))</span>"</span>)
        _swizzled_viewWillAppear(animated)
    }
}
</code></pre>
<p>When <code>viewWillAppear</code> is called, the system runs the code inside <code>_swizzled_viewWillAppear</code> instead. In this function, a recursive call is made which ends up executing the implementation of the original <code>viewWillAppear</code>. In short, when the view is about to be displayed, the program prints a log, for example, <code>Enter screen: LoginViewController</code> and does what it is supposed to do.</p>
<p>Notice:</p>
<ul>
<li>In order to swizzle successfully, the methods must be dynamically dispatched via message. So, we explicit declare it with <code>dynamic</code> keyword. Of course, at times you don't necessarily need that keyword to make it dynamic :D.</li>
<li>The swizzling action for a pair of methods should only run once.</li>
</ul>
